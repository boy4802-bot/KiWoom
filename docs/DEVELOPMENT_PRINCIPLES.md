# KiWoom 자동매매 — 개발 원칙

본 문서는 키움증권 REST API 기반 자동매매 프로그램 개발 시 **반드시 지켜야 하는 원칙**입니다.  
모든 단계는 `roadmap.md`의 트리 구조를 따르며, 단계 완료 시 이 문서와 로드맵을 함께 갱신합니다.

---

## 핵심 원칙 (9가지)

### 1. 전체 로드맵 구성 후 단계별 진행

- `roadmap.md`에 정의된 **상위 단계 → 하위 단계** 순서를 지킨다.
- 현재 단계가 완료·검증되기 전에는 다음 단계 코드를 작성하지 않는다.
- 범위가 커지면 로드맵에 하위 항목을 추가한 뒤, 해당 항목부터 다시 진행한다.

### 2. 단계별 완료 후 검증

각 단계마다 아래를 수행하고, `roadmap.md`의 **검증** 열에 결과를 기록한다.

| 검증 유형 | 내용 |
|-----------|------|
| 기능 검증 | 해당 단계 목표 API·기능이 정상 동작하는지 |
| 모의투자 검증 | 실거래 전 `https://mockapi.kiwoom.com` 환경에서 주문·조회 테스트 |
| 회귀 검증 | 이전 단계 기능이 깨지지 않았는지 |
| UI 검증 | Streamlit 화면에서 설정·실행·결과 확인 가능한지 |

검증 실패 시: 원인 기록 → 수정 → 재검증 → 통과 후에만 다음 단계.

### 3. 변수 및 파일명 — 짧고 직관적으로

- **파일명**: 소문자, 언더스코어, 역할 중심 (예: `auth.py`, `ord.py`, `strat_ma.py`)
- **변수명**: 약어 허용, 의미가 드러나게 (예: `tkn`, `acc_no`, `qty`, `prc`)
- **금지**: `data1`, `temp`, `test123` 등 의미 없는 이름

### 4. 단계별 완료 후 GitHub 커밋

- **한 커밋 = 로드맵의 한 하위 단계** 완료 시점.
- 커밋 메시지: `[단계번호] 간단한 설명` (예: `[1.2] OAuth 토큰 발급`)
- `.env`, `appkey`, `secretkey` 등 **비밀값은 절대 커밋하지 않는다**.
- `main` 브랜치는 검증 통과한 코드만 반영한다.

### 5. 진행 상황은 roadmap.md로 관리

- 단계별 상태: `[ ]` 대기 / `[~]` 진행 / `[x]` 완료
- 원칙·설계 변경은 `docs/DECISIONS.md`에 기록 (필요 시 생성).

### 6. 로드맵은 트리 형태로 관리

- 들여쓰기와 번호로 **부모–자식 관계**를 명확히 한다.
- 하위 항목은 상위 항목 없이 완료 처리하지 않는다.

### 7. 스크립트별 해설서(비전문가용)

- `foo.py` 작성 시 **`foo_guide.md`** 를 같은 경로에 둔다.
- 해설서: 하는 일, 실행 시점, 바꿔도 되는 설정, 자주 나는 오류.
- 코드 주석은 **왜(Why)** 위주, 한국어로 작성한다.

### 8. 목적: 자동매매 + 전략 등록

- **등록 가능한 매매 전략**이 핵심이다.
- 전략은 플러그인 형태로 추가·교체 (`strategies/` 폴더).
- 실주문 전 **모의투자** 경로를 반드시 제공한다.

### 9. 배포 가능한 실행 프로그램

- 최종 산출물: **Windows 64bit 실행 파일** (PyInstaller).
- Python 3.11+ **64bit** 기준.
- 배포물: exe, 설정 템플릿, `README.md`.

## 기술 스택

| 항목 | 선택 |
|------|------|
| API | 키움 REST (`api.kiwoom.com` / 모의 `mockapi.kiwoom.com`) |
| 인증 | OAuth2 `au10001` |
| 언어 | Python 3.11+ (64bit) |
| UI | Streamlit |
| HTTP | httpx |
| 실시간 | WebSocket |
| 패키징 | PyInstaller |
| 참고 | `키움 REST API 문서.pdf` |
